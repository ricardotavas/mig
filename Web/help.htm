Ajuda da impressora

Sistema
  • Driver em uso
    • Escolha o drive que será usado pela impressora para gravar todos os arquivo do sistema e também os arquivos de relatórios, o drive padrão é a memória Flash interna, tendo também a opção de usar um SDCARD.

  • Ajustar data e hora do sistema
    • Selecionando esta opção ao confirmar o envio a data e a hora do sistema serão enviadas para ajustar a impressora, esta informação é impressa no ticket do cliente.

  • Auto teste de impressão de ticket
    • Força a impressora a emitir tickets do departamento 1 em intervalos de 1 segundo para teste do sistema.

  • Tempo de vida útil do ticket
    • Este tempo impede que a senha seja fechada indevidamente ao gerar o relatório no fechamento, caso a senha ainda não tiver sido atendida, o ticket ainda ficará aberto para que possa ser chamado e somente deverá aparecer no relatório do dia seguinte.

  • Imprimir protocolo de atendimento
    • Esta opção é para integração com outros sistemas, fazendo com que este servidor MIG solicite um número de protocolo de atendimento a um servidor de protocolos do cliente para imprimir este número no rodapé do ticket do cliente.

  • Endereço IP do servidor de protocolo
    • Quando a opção imprimir número de protocolo estiver habilitada é necessário informar o IP do servidor de protocolo do cliente para que este servidor MIG possa solicitar o número para impressão.

  • Controle de funcionamento
    • Permite controlar o horário de funcionamento do sistema de senhas, o sistema de senhas pode ser composto de teclados de chamada, painéis eletrônicos e acionadores, configurando estes parâmetros todos podem ser desligados no final do dia automáticamente.
    Habilitar controle - Habilita este controle de horário de funcionamento.
    Horário para ligar e desligar - Configure os horários inicial e final do atendimento, o sistema entrará em funcionamento somente dentro deste intervalo.

  • Controle do relatório
    • Horário para fechar o atendimento - Configure o horário para fechar o atendimento, este horário deve estar dentro do intervalo de funcionamento do sistema, caso o sistema estiver desligado o fechamento não será realizado. O fechamento do atendimento é o momento em que os tickets são encerrados para liberar espaço para novos tickets, somente os tickets ainda com tempo de vida serão preservados.
    Atenção - A impressora de tickets, grava todas as senhas chamadas em memória respeitando os ranges de numeração programados na configuração do atendimento, e não faz gravação sobreposta em tickets que ainda não foram encerrados ou cancelados para preservação dos dados para o relatório.
    Gravar arquivo .CSV de relatório - Habilite esta opção para que o arquivo do relatório seja criado após o fechamento do atendimento. Ao habilitar esta opção fique atento ao espaço de memória disponível para gravar arquivos, caso o espaço seja insuficiente será necessário formatar a memória para liberar espaço. Note que com esta opção ativada o sistema não será fechado enquanto não conseguir gravar o arquivo.

  • Modo de uso da impressora de tickets
    • Modo Servidor - Armazena todas as informações do atendimento, imprime senhas, responde a solicitações de painéis de senhas, teclados e softwares de chamada e gera arquivos de relatório.
    • Modo Cliente - Solicita senhas para um outro dispositivo que esteja configurado como servidor, e apenas imprime o número do ticket recebido para o cliente em outro local.

  • Endereço IP do servidor (modo cliente)
    • Em modo cliente, a impressora depende de uma outra impressora que esteja configurada como servidor, informe aqui qual o IP deste servidor.

  • Informações de fila
    • Enviar informações de fila - Habilita o envio de informações de número de pessoas na fila e tempo de espera em cada atendimento. Esta informãção é utilizada por softwares de chamada para atendimento.

  • Interface de envio
    • Escolha a interface - Indica qual meio de comunicação será utilizado para enviar informações de fila ou requisições de ticket quando em modo cliente. Esta configuração não se aplica a respostas do modo servidor.

    Departamentos

  • Escolha do departamento
    • Escolha um departamento, esta impressora fornece 12 opções, onde cada um atende a uma determinada faixa de números de tickets, que serão atendidos por usuários numerados de 1 a 99, estude o melhor modo de atender seus clientes.

  • Status do botão na impressora
    • Habilita o departamento e permite que os usuários do sistema possam chamar os números de tickets, quando desabilitado nenhum número será chamado neste departamento.

  • Nome
    • Nome do departamento, este nome será exibido no ticket entregue ao cliente.

  • Letra do ticket
    • Caso o painel eletrônico de chamada possua o caractere alfa-numérico, você poderá definir aqui uma letra para identificar com mais clareza o departamento.

  • Faixa de números do ticket
    • Define o intervalo de números que o departamento irá atender, os tickets serão emitidos em ordem crescente e sequêncial do número inicial ao final, caso a numeração chegue ao último número ela será reiniciada no primeiro número no próximo ticket, porém nenhum número repetido será emitido, para que o sistema volte a emitir um mesmo número ja usado, antes deve ser gerado um relatório. Os relatórios são gerados automáticamente 1 vez ao dia para controle de histórico de atendimento.

  • Encerrar ticket automático
    • Estas opções permitem que o ticket seja finalizado automáticamente ao chamar um próximo número ou ao gerar o relatório, quando o usuário esquecer de pressionar o botão encerrar, finalizar o ticket significa gravar o horário de término de atendimento e mudar o status para "atendido", estas informações são mostradas no relatório.

  • Esquema de chamada
    • Define a proporção de chamada entre clientes normais e preferênciais, a alteração pode ser feita a qualquer momento para agilizar o atendimento.

  • Ajudar departamentos
    • Permite escolher escolher um ou dois departamentos para ajudar, ou ajudar o departamento de maior fila.

    • Ignorar dptos de ajuda e procurar maior fila - Ajuda sempre a maior fila entre os dptos, mesmo que exista um dpto de ajuda configurado.

  • Esquema de ajuda
    • Quando configurado um dos esquemas de ajuda, define a proporção de chamada entre clientes locais e do outro departamento, a alteração pode ser feita a qualquer momento para agilizar o atendimento.

    • Ajudar quando acabarem os tickets do dpto local - Ajuda o outro departamento escolhido somente quando acabarem todos os tickets na fila local, isto garante que toda a fila local vai ser atendida antes de ajudar.
    • Antes de ajudar, verificar o tempo de espera local - Com esta opção ativada o sistema verifica os tempos de espera no dpto local e no dpto de ajuda, para garantir que o cliente na fila local que chegou mais cedo sempre seja atendido antes de ajudar o outro dpto. Ou seja, somente ajuda um cliente de outro dpto se este chegou mais cedo.
    • Alternar o esquema de ajuda por usuários - O padrão é alternar entre chamada local e chamada de ajuda entre so deptos, ou seja se estiver com esquema 1x1, chama um ticket do dpto1 e outro do dpto2, não importa qual o atendente. Com esta opção, o sistema alterna os tickets para cada usuário, ou seja, se estiver no esquema 1x1 e o usuário 3 chamou um ticket local, a próxima chamada deste usuário vai ser ajudando o outro departamento, independente das outras chamada já feitas pelos outros usuários do mesmo dpto.

  • Usuários do departamento
    • Selecione quais dos usuários deverão atender a este departamento, ao incluir um usuário ele automáticamente é retirado de outro departamento ao qual pertencia antes.

  • IP do display de chamada
    • Quando estiver usando acionadores de R.F. inteligentes para chamar a senha será necessário fazer esta configuração, como estes acionadores se comunicam diretamente com a impressora o endereço IP do painel eletrônico de chamada deverá ser informado.

  • Preferêncial do departamento
    • Permite que o botão da impressora relativo a este departamento, seja usado para emitir senhas preferênciais de outro departamento, neste caso nenhuma das configurações feitas acima serão usadas.

    Box do usuário

    Esta opção permite montar uma relação entre usuário e número do box de atendimento, permitindo que usuários de um sistema de atendimento com muitos setores, possam chamar um mesmo número de box em seus respectivos displays de chamada. Ex: Usuário 1 atende no Box 10 e Usuário 2 atende também atende no box 10 porém de outro setor.

  • Selecione o usuário
    • Permite selecionar qual usuário terá o box configurado.

  • Configure o box do usuário
    • Permite escolher qual o box do usuário previamente escolhido, a configuração de fábrica é o número do box igual ao número do usuário. Clique no botão Confirmar para alterar.

  • Configuração atual: Box X Usuário
    • Exibe uma lista de todos os usuários e respectivo box.

    Ticket

  • Cabeçalho com logotipo
    • Habilite a impressão do logotipo no cabeçalho do ticket, caso ainda não exista nenhum arquivo gravado o logotipo impresso será o padrão Wyma, verifique antes se ja foi gravado o arquivo de sua empresa o nome do arquivo de logotipo aparece na listagem de arquivos como logo.bmp.

  • Letra do ticket
    • Habilite a impressão da letra do departamento antes do número do ticket, antes de habilitar esta opção verifique se a letra ja foi configurada no departamento e se o painel eletrônico pode mostrar a letra.

  • Comprimento do número
    • Determine aqui quantos digitos o número do ticket vai exibir, deve ser igual ao painel.

  • Altura do número
    • Escolha o tamanho do número do ticket, existem dois tamanhos: 5mm e 15mm.

  • Corte do papel
    • Defina como vai ser cortado o papel após imprimir o ticket.

  • Número de vias
    • Defina quantas vias de ticket devem ser impressas.

  • Linhas do cabeçalho
    • Opções de mensagem - Escolha qual informação vai ser exibida em cada linha do cabeçalho.
    Mensagem - Quando selecionado Mensagem nas opções, este será o texto impresso no ticket. Normalmente é o nome da empresa, a terceira linha do cabeçalho será usada para informar que o ticket é preferêncial, quando for este tipo de ticket.

  • Linhas do rodapé
    • Opções de mensagem - Escolha qual informação vai ser exibida em cada linha do cabeçalho.
    Mensagem - Quando selecionado Mensagem nas opções, este será o texto impresso no ticket.

    Rede

    Configure os parâmetros de sua rede intranet, caso tenha um servidor DHCP, a impressora poderá ser automáticamente configurada em sua rede. Para visualizar o IP pressione as teclas 1 e 12 ao mesmo tempo durante 3 segundos.

    Versão

    Verifique a versão de seu dispositivo. A opção autenticação permite proteger suas informações, grave uma senha de acesso para acessar o servidor.

    Personalizar

    Crie uma apresentação com imagens para a tela do display LCD, salve dois arquivos bitmaps monocromáticos em formato 128x56 e deixe as imagens passarem informações para os clientes. Use uma imagem monocromática para o logotipo do ticket.

    Arquivos

    Faça o download de relatórios de atendimento e salve em seu computador, os arquivos são criados em um formato que permite abrir o arquivo em programas de planilha eletrônica como o Excel da Microsoft.

    Tempo Real

    Visualize as informações básicas de atendimento em tempo real.

    Formatar

  • Formatar o sistema de arquivos
    • Os arquivos de imagens do sistema e relatórios são gravados em um sistema de arquivos, quando formatamos o sistema de arquivos todos os arquivos são apagados e o sistema é reconstruído para otimização, quando devemos fazer isto:
    1 - Formate o sistema de arquivos antes do primeiro uso.
    2 - Quando o sistema de arquivos FAT estiver corrompido. Identificando um sistema corrompido: listagem de arquivos com nomes irreconhecidos ou excessiva demora na leitura ou gravação.
    3 - Formate o sistema de arquivos para uma melhor performance de leitura dos arquivos gravados, assim que perceber um desempenho mais baixo na abertura dos arquivos de relatório, salve todos os arquivos em seu computador, formate o drive e grave novamente somente os arquivos de imagens do sistema, este procedimento otimiza o tempo de gravação e leitura dos arquivos e libera espaço no dirve. Quando estiver utilizando a memória Flash interna a capacidade será bem menor e o desempenho mais baixo que o cartão SD.

  • Inicializar a numeração das senhas
    • Inicializa a numeração dos tickets de todos os departamentos, mas cuidado esta opção descarta todos os tickets em atendimento, todas os tickets que estiverem na fila de espera serão eliminados, se não for o primeiro uso, certifique-se de que não há mais clientes para atender.

    Atualizar

    Permite atualizar o firmware da impressora por uma versão mais recente. Para verificar a versão do firmware clique na página inicial em Versão.
